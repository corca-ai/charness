# Agent/Human Lens

`ideation` should treat both humans and agents as first-class parts of the
concept.

## Agent-First Questions

- should this concept be agent-native from the start?
- what should exist as CLI, skill, or API before a web UI exists?
- what capability boundaries, runtime limits, or tool assumptions constrain the design?
- where will agents need structured interfaces rather than ad-hoc screens?

## Interface Questions

- even if UI comes later, what interface qualities still matter now?
- what human action or judgment must stay visible and legible?
- where will poor interface design break trust or adoption?

## Human Understanding Questions

- what cognitive load does this concept impose?
- what incentives, habits, or social behaviors shape adoption?
- what misunderstanding is the user most likely to have?

## Rules

- agent-first is a prioritization rule, not an excuse to ignore interface quality
- human factors and agent factors should be considered together when the system
  includes both
- if the concept assumes "the agent will handle it", test that claim against
  actual runtime or workflow constraints
