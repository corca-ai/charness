# Debug Memory

The debug artifact is not busywork. It is the codebase's debugging memory.

## Why It Matters

- future sessions can read near-matches before starting over
- recurring failures become visible as patterns
- the final root cause and prevention remain legible after the pressure is gone

## Artifact Rule

Use the repo's existing incident, debug-note, or troubleshooting surface when
one exists. If none exists, create the smallest durable artifact that matches
the repo's document style.

Do not create a fresh format every time.
