# Spec Boundary

`spec` is downstream of `ideation`.

## What Spec Inherits

From `ideation`, inherit:

- the core concept
- the wedge or system shape
- the important assumptions
- the durable world model when present

## What Spec Adds

`spec` adds:

- the current implementation slice
- fixed decisions
- probe questions
- deferred decisions
- explicit non-goals
- success criteria
- acceptance checks
- first or next implementation slice

## Send Back To Ideation When

Send work back when:

- the core user or operator is still unclear
- there is no stable wedge or concept shape
- the biggest assumption is still concept-defining
- the current document reads like brainstorming rather than a decision frame

Normal implementation learning does not require a trip back to `ideation`.
Concept-defining drift does.
