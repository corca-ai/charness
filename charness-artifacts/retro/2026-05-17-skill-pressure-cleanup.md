## Waste

- I initially over-optimized pressure terms by changing exact schema vocabulary
  (`gather_provider.<source>.mode`) into a softer route phrase. The fresh-eye
  critique caught that this was semantic drift, not harmless compression.
- Updating the quality artifact after the full gate took extra loops because
  its validators require exact runtime-signal and inventory-consumption
  evidence fields.

## Critical Decisions

- Keep Charness' default skill ergonomics policy opt-in, but enable all
  low-noise rules in this repo's `.agents/quality-adapter.yaml`.
- Exempt `## References` from core pressure counting because it is link
  inventory, while recording a future guard if it starts carrying workflow prose.
- Ignore inline code spans for mode/option pressure so schema keys and canonical
  taxonomy terms can remain exact.

## Expert Counterfactuals

- Daniel Jackson would have separated "term pressure" from "domain vocabulary"
  earlier; the concept boundary is avoidable branching, not every occurrence of
  a word like mode.
- Atul Gawande would have turned the quality artifact validator failures into a
  short checklist before rewriting the artifact: runtime source, hot spots,
  coverage gate, evaluator depth, and declared inventory fields.

## Next Improvements

- workflow: when reducing pressure terms, first classify each term as casual
  branch language, schema literal, or canonical taxonomy before editing.
- capability: add a future low-noise rule that `## References` remains link
  inventory rather than hidden workflow prose if that drift appears.
- memory: preserve this lesson in recent retro lessons so future skill
  compression does not trade exact contracts for a cleaner heuristic score.
