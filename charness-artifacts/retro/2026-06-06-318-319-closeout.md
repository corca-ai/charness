## Sibling Search

Transferable pattern: a line-field regex `^…Field\s*:\s*(.+?)\s*$` with
`re.MULTILINE` borrows the *next* line as the value when the field is blank
(because `\s*` after the colon spans the newline). Four-axis scan across the
achieve/issue/quality field parsers:

- `goal_artifact_closeout_evidence._EVIDENCE_LINE` (`Retro:`/`Host log probe:`/
  `Disposition review:`): has the pattern — but **fails safe**. A blank field
  that borrows the next line yields a wrong path that then fails the
  existence/binding check → the gate *refuses*, never falsely satisfies.
- `goal_artifact_timebox.py` (`Timebox:`/`Activation time:`/`Closeout reserve:`/
  `No safe next slice:`/`Early close rationale:`): has the pattern — **fails
  safe**. A borrowed value either fails duration/ISO parsing (treated absent) or
  registers an early-close reason that *adds* the early-close-report requirement
  (stricter, not a bypass).
- `goal_artifact_closeout_delegation._ORCHESTRATOR`/`_MODE`: **was** exploitable
  (the orchestrator check is pure presence, so any borrowed line satisfied it) —
  **fixed this run** (same-line `[ \t]` capture + regression test).

Decision: the delegation case was the only exploitable instance and is fixed;
the siblings are all fail-safe (downstream validation or stricter-requirement
direction), so no follow-up issue is warranted. A same-line hardening of the
fail-safe siblings is optional robustness, deliberately out of this goal's scope.
