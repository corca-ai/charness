## Goal

Durably resolve the six OPEN follow-up issues in the 306–317 range and close
them on `main`, then cut a release:

- **#306** — stop the scheduled mutation gate's changed-line signal from
  generating ~2 self-healing auto-issues/day. Make subprocess-only CLI-script
  coverage reliably captured (so the recurring class stops reading as 0%) **and**
  cover that script class directly with in-process tests. The gate stays
  blocking — noise is removed by making the signal honest, not by demoting it.
- **#311** — make `setup`'s inspector flag an existing `AGENTS.md` as stale when
  it lacks the #303 adapter-first subagent-reviewer rule, and add that rule to
  the required-snippet set. `setup` keeps its never-rewrite-existing-body design;
  the operator is *told* the file is stale rather than having it rewritten.
- **#314** — wire the fast structural checkers (`validate_skill_ergonomics.py`,
  `check_boundary_bypass_ratchet.py`) into the per-slice surface `verify_commands`
  **and** reconcile the per-slice aggregate (`run_slice_closeout.py`) with the
  literal git pre-commit hook (`staged_commit_gate_plan.py`) so both commit-
  boundary paths run the same fast gate subset. Generalizes the closed #307.
- **#315** — seed visible closeout-evidence placeholders into the `achieve` goal
  artifact template (retro, host-log probe, disposition review, Auto-Retro
  dispositions) so an active agent sees the obligation from the start — while
  keeping the placeholders unable to satisfy the `complete` gate (literal
  `TODO`/`<path>`/`TBD` still fails).
- **#316** — make `achieve`'s external-publication/apply approval boundary
  explicit: approval is scoped to the phase/bundle that requested it; after an
  approved publish/CI/apply lane, done-early test-only quality continuation is
  **local by default**; per-slice remote CI is not assumed.
- **#317** — make `setup` seed a compact meaningful-slice commit-discipline block
  in greenfield `AGENTS.md` (commit meaningful work slices as they finish; treat
  meaningful `charness-artifacts/` changes as repo state; do not report a goal
  done while meaningful work is uncommitted unless deferral is explicit), and have
  the setup inspector flag an `AGENTS.md` that carries Charness goal routing but
  no commit discipline. Same "tell, don't rewrite" pattern as #311: existing
  bodies are not rewritten. Detailed rationale stays in setup references.

Ending: cut a release (version bump + publish) as the final slice; the final
push auto-closes the `Close #N` carriers. (Closeout correction: charness has no
push-triggered CI, so the local `--release` gate is the bundle proof, not a
remote CI run — see `## Final Verification`.)
