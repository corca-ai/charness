## Goal

Close the mutation-testing backlog cluster as one auditable run:

1. **#262 — runner defensive restore.** Make `scripts/run_cosmic_ray_mutation.py`
   defensively restore the config's `module-path` files on *every* `cosmic-ray
   exec` outcome (clean / non-zero / internal-timeout) AND on wrapper-process
   interrupt (SIGINT/SIGTERM), so a killed exec never leaves a mutated source
   file in the working tree. The list comes from the config the runner already
   reads; the restore preserves the original exit/failure semantics.
2. **#261 — latent survivors in the coordination-cues achieve trio.** Triage and
   kill the ~60 non-equivalent mutation survivors in
   `goal_artifact_closeout_evidence.py`, `goal_artifact_coordination_floors.py`,
   and `check_goal_artifact.py` with targeted helper tests; name (do not chase)
   the known-equivalent mutants.
3. **#219 — stale auto-filed mutation regression.** Determine the current-`main`
   state of the regression (FAIL 66.1% at the old commit `59841e0`; survivors in
   `scripts/artifact_closeout_lib.py` and `scripts/validate_rca_ledger.py`).
   Either kill its remaining real survivors or document it as
   resolved/equivalent-noise with evidence.

Stage `Close #262` / `Close #261` / `Close #219` on the default-branch closeout
commit body. `achieve` shapes and audits; it does not push or close out-of-band.

## Boundaries

- **#262 restore reads the *current* config.** `cosmic-ray.toml`'s `module-path`
  is rewritten in place by the mutation sampler, so the restore must read the
  list from the config the runner is given — never hardcode it (today it is
  `["scripts/control_plane_lib.py"]`, but it varies per sampled run).
- **Restore scope + mechanism.** Restore is scoped to the config's tracked
  `module-path` files only, reverting them to their committed (HEAD) / pre-exec
  state via `git checkout`/`git restore` — the proven mid-session #260 mitigation
  and the issue's suggested mechanism. Cosmic Ray already requires a clean
  baseline for those files, so a dirty working tree for module-path files is
  out-of-contract for a mutation run; the restore reverts them and emits a
  visible log line rather than clobbering silently. (Clobber edge confirmed
  acceptable by the plan critique below.)
- **Restore fires on all kill paths.** try/finally covers every exec outcome; a
  SIGINT/SIGTERM handler covers an external `timeout`/Ctrl-C killing the wrapper.
  The restore is defensive cleanup — it must NOT swallow the original failure or
  change the wrapper's exit code semantics.
- **Survivor work adds tests, not assertion weakening.** #261/#219 helper tests
  must add real kills without duplicating existing coverage or relaxing existing
  assertions; watch the duplicate/length gate (cheap pressure sample per
  test-adding slice).
- **No release surface.** This run touches a runner script + tests only — no
  version bump, no install-manifest edit.

## Plan Critique Findings

Reviewer provenance: a bounded fresh-eye plan critique (separate agent context,

**Verified facts folded into the plan:**

- The runner today only *resolves* the config path — it does **not** parse
  `module-path`. So slice 1 adds TOML parsing of `module-path` (folded into the
  Slice 1 row and the #262 restore boundary). Not a gap; the boundary already
  implied "reads the config," now explicit.
- `scripts/artifact_closeout_lib.py` carries `from __future__ import
  annotations`, so its `str | None` `BitOr` survivors are **equivalent by
  construction** (annotation strings are never evaluated). This confirms #219's
  equivalent-noise hypothesis is the likely triage outcome — still verified, not
  assumed (folded into the Slice 2 note + Non-Goals).
- `59841e0` (the #219 commit) is **~140 commits behind HEAD**, so the staleness
  is genuine and the "triage current-`main` state before fixing" gate is
  verified-necessary, not defensive over-caution.

**Over-worry raised but NOT folded** (slice-time impl details the test plan /
acceptance lines already gate — recorded so the run does not rediscover them):

1. *SIGINT/SIGTERM re-raise mechanism.* The handler must restore, then reset to
   the default disposition and re-raise so the wrapper dies with `128+signum`
   (not a bare `sys.exit(0)` that swallows the signal). Gated by Slice 1's
   "signal path" test.
2. *Restore-itself-failing / multi-file partial mutant.* When `module-path` is a
   list, a `git checkout` of one file may fail (deleted file, git not on PATH).
   Handle best-effort over the list (log + continue); carry as a Slice 1
   non-claim rather than discover mid-slice.
3. *Git-axis justification, not conclusion.* The `single-point` call for
   git-restore is correct, but the honest justification is "this runner's
   contract assumes a git working tree (cosmic-ray baseline already enforces a
   clean committed tree)," not "git is not an axis." Conclusion stands;
   justification tightened here.
4. *Stale-green #219 closeout.* If triage finds #219 already green on current
   `main`, `Close #219` is **still staged** with a "resolved-by-drift; evidence:
   scoped re-run clean" note — never silently dropped. (User Acceptance already
   covers this; flagged so a stale-green #219 is not treated as "nothing to
   close.")
