## Boundaries

- **Slice A (warn tier):** advisory only — `check_python_lengths.py` prints a
  `WARN` line for files in `[warn, limit]` but keeps exit 0; `> limit` still
  hard-fails unchanged. **The line MUST start with `WARN:` (or `WARN `)** — else
  `run-quality.sh`'s passing-gate surface filter
  (`^(WARNING|WARN|WEAK|ADVISORY)(:|[[:space:]])`, ~line 296) captures the
  advisory to the log but never shows it, silently defeating the warn tier.
  Warn thresholds (file length): skill helpers
  (`skills/public|support/*/scripts/*.py`) 330/360; repo `scripts/*.py`
  432/480; `tests/**/*.py` 720/800. The exported copy
  `plugins/charness/scripts/check_python_lengths.py` is a synced surface —
  regenerate it (mutate → **sync** → verify). Invocation stays via
  `run-quality.sh` (`check-python-lengths`) and pre-push; do **not** add to
  pre-commit.
- **Slice B (#232):** fix lives in `skills/public/issue/` (`issue_tool.py` /
  `issue_runtime.py` + `SKILL.md` guidance). Body must reach the backend via
  stdin or a temp file, never interpolated into a shell command string. Note:
  `issue_tool.py` has **no `create` subcommand today** — `SKILL.md` directs the
  agent to call `gh issue create` directly. The fix likely adds a `create`
  subcommand with `--body-file` (parallel to the existing `close-with-comment`),
  plus guidance against inline `--body`. `issue_tool.py` is already 352 lines
  (inside the 330 warn band), so a new `issue_create.py` module (parallel to
  `issue_close.py`) is **expected, not contingent** — `issue_tool.py` must stay
  ≤360. Provider-agnostic (works for `gh` and any host-mediated backend).
- **Slice C (#244):** add an adapter-gated, opt-in `find_skills_routing`
  host-hooks intent in `host_hook_install_lib.py`, pointing at the installed
  plugin's `session_start_find_skills.py`, matcher `startup|resume|clear`,
  parallel to usage-episodes. Keep the dumb-hook / skill-owns-routing split.
- **Slice D (#245):** de-duplicate SessionStart hooks by *logical identity*
  (script basename / intent) rather than exact absolute-path command string, so
  two checkouts do not double-install. Same file/seam as C
  (`_entries_match_command`); must not break the legitimate single-install case
  or the Codex TOML path (`host_hook_codex_toml_lib.py`).
- **Phase discipline:** mutate → sync (plugin export + generated surfaces) →
  verify (validators) → commit per slice. Meaningful `charness-artifacts/`
  changes commit with their slice.
