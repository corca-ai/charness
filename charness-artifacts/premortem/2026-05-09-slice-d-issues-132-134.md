## Success Criteria

- `close-with-comment` runs default `gh` shape and adapter-templated
  shape with body-file substitution; partial close-failure surfaces a
  retry-safe structured error.
- 5 contract additions land in skill `references/` files with exact key
  phrases enforced by citation chain tests.
- Downstream materialization drift contract is documented in the home
  that downstream tooling authors already read.
- All quality gates and the full pytest sweep stay green; plugin
  manifests stay synced.

## Out of Scope

- Runtime validators for the prose contracts (e.g., enforcing that bug
  close comments contain `JTBD:`). Citation-chain tests are the
  documented enforcement pattern; runtime shape-of-text checks would
  block honest phrasing variants without catching new failure modes.
- Implementing the Ceal-side materialization gate. That fix lives in
  Ceal; Charness only owns the contract docs that downstream tooling
  can point at.
- Generalizing `_resolve_op` into a shared helper used by both `issue`
  and `release` skills. Three callers (issue create+view+close+comment,
  release create+view+auth) is still acceptable per CLAUDE.md
  premature-abstraction discipline; revisit if a fourth skill needs
  the same shape.
