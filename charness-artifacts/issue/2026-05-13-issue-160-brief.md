# Resolution Brief — corca-ai/charness#160

**Classification**: feature

**Reporter JTBD**: maintainer wants to configure Charness adapter triggers by
surface id (not raw glob), get auditable trigger behavior (including config
error when a configured surface id does not resolve in the manifest), and have
a portable fixture shape they can copy across skills.

**Proposed boundary**:
- in scope:
  - factor `resolve_trigger_surfaces(manifest, configured_ids)` in
    `scripts/surfaces_lib.py` that returns `(hits, unresolved)` and surfaces a
    broken-config signal when configured ids are not declared in
    `.agents/surfaces.json`
  - rewire `skills/public/retro/scripts/check_auto_trigger.py` and
    `skills/public/release/scripts/check_real_host_proof.py` (sibling found
    via mental-model scan — same silent-miss pattern at L63-67) to use the
    helper; both emit explicit `configuration_status: "broken"` + nonzero exit
    (mirrors existing missing-manifest path)
  - portable 3+1 fixture coverage in `tests/quality_gates/`: true positive /
    unrelated-doc false positive / clean false case / typo'd-surface-id
    config-error case. Retro has 3/4 today; release has 0/4.
  - new `docs/conventions/surface-driven-adapter-triggers.md` codifying the
    contract. Cited from `retro/references/adapter-contract.md` and
    `release/references/install-surface.md`.
- out of scope:
  - generalizing into a single shared trigger CLI across skills
  - extending to setup-adapter surface triggers (no analogous field today)
  - retroactively migrating `auto_session_trigger_path_globs` entries to
    surface ids (handoff item 5 separates that as dogfood-driven work)

**Non-goals**:
- producing a Cautilus eval (deterministic gates own closeout; no
  prompt-affecting surface change)
- changing the existing `match_surfaces` shape or surface manifest schema

**Acceptance checks**:
- `test_retro_auto_trigger_*` and new `test_release_real_host_proof_*` cover
  the 3+1 fixture shape; previously-passing tests stay passing
- new test demonstrates that a typo'd configured surface id produces a
  config-error payload naming the unresolved id, not `triggered: false`
- `./scripts/run-quality.sh` (or local-equivalent) green after sync
- close comment captures Boundary + Resolution brief marker + implementation
  summary

**Open decisions**:
1. Sibling bundling: include `release/check_real_host_proof.py` in the same
   commit (recommended) or keep it retro-only and file the release sibling
   as a separate issue.
2. Failure shape: emit broken-config as nonzero exit + JSON on stderr
   (recommended, mirrors existing missing-manifest path) or zero exit +
   `configuration_status: "broken"` field only.

**Autonomous vs pause**: pausing — two non-empty open decisions.
