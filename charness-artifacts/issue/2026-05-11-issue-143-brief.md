# Resolution Brief — corca-ai/charness#143

**Classification**: `feature` (label: `feature request`; proposes adding a
new workflow step to the `charness:issue` skill — no contract divergence to
fix)

**Reporter JTBD**: when asking an agent to resolve a non-trivial GitHub
issue, the operator wants the agent to surface its issue reading and proposed
resolution boundary before implementation, so product/scope disagreements
can be caught while they are still cheap.

## Proposed boundary

Insert a new step between current step 5 (ordering) and current step 7
(design/implement) in `skills/public/issue/SKILL.md`. The new step runs only
for `feature` and `deferred-work` classifications. It emits an inline
**Resolution Brief** with: classification, JTBD, proposed product/workflow
boundary, non-goals, acceptance checks, open decisions/tradeoffs, and an
autonomous-vs-pause intent line. Non-empty `open decisions` always forces a
pause; the adapter setting and invocation flag only control the
empty-decisions case.

In scope:

- `feature` and `deferred-work` resolution paths
- adapter schema gains `feature_brief_pause` (`on-open-decisions` default,
  `always`, `never`)
- invocation flags `--auto`/`--pause` override per call
- brief persistence to `charness-artifacts/issue/<date>-issue-<n>-brief.md`
  when the brief pauses
- trivial-feature short-circuit symmetric to the existing trivial-bug
  short-circuit
- transcript audit script + fixtures verifying mutation does not precede
  brief
- close-comment shape for feature/deferred-work gains `Boundary` and
  `Resolution brief: <path>` lines

Out of scope (Non-goals):

- adding a brief for `bug` (causal review already covers it)
- changing `issue new` flow
- making brief persistence mandatory for non-pause cases
- adding a runtime mutation-blocker (audit is post-hoc)

## Acceptance checks

1. `SKILL.md` and `references/resolve-flow.md` carry the new step with
   explicit pause vs continue rule.
2. A new reference `references/resolution-brief.md` documents the template,
   precedence table, persistence rule, trivial short-circuit, range-resolve
   rule, and close-comment coupling.
3. `resolve_adapter.py` parses and exposes `feature_brief_pause`.
4. `issue_tool.py resolve-invocation` strips `--auto`/`--pause` flags and
   reports `brief_pause.{adapter, flag_override, effective}` for the agent.
5. `issue_tool.py brief-path` resolves the dated artifact path.
6. `audit_brief.py` plus four fixtures (good-feature-pause, good-feature-
   trivial, good-bug, bad-jumped-to-mutation) verify the audit contract.
7. Dogfood acceptance evidence in `docs/public-skill-dogfood.json` and the
   scaffold in `scripts/public_skill_dogfood_lib.py` carry the new contract.
8. Self-test: applying this skill to #143 produces a visible brief and the
   pause (satisfied by this very session).

## Open decisions (resolved with user approval)

1. **Skip signal**: adapter setting `feature_brief_pause` plus invocation
   flag `--auto`/`--pause` override (option d).
2. **Brief persistence**: artifact only when the brief pauses (option c).
3. **Acceptance fixture shape**: standalone audit script with fixtures, not
   wired into CI (option b).
4. **Trivial-feature short-circuit**: kept (option a), symmetric to
   trivial-bug.

## Autonomous-vs-pause intent

Paused for user input on the four open decisions above. Continued
implementation after user replied "제안 수용" accepting the recommended
options (d, c, b, a).

## Critique outcome

Resolution critique surfaced one cheap-prevention item — a symmetric
misclassification guardrail covering the inverse direction (feature → bug,
feature → question/decision-needed). Added to `SKILL.md` guardrails before
commit. Angle 2 (brief becoming theater) closed as over-worry because the
close-comment `Boundary` field is the durable substantive check and any
machine "substance" heuristic would push agents back into misclassification
or short-circuit abuse.
