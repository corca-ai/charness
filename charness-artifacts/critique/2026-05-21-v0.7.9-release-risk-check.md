# v0.7.9 Release Risk Check

Date: 2026-05-21
Scope: publish the current `v0.7.8..HEAD` maintenance delta as `charness`
patch release `v0.7.9`.
Fresh-Eye Satisfaction: short scoped release critique for routine patch
hygiene; no standalone fresh-eye pass was run before this publish request.

## Act Before Ship

- Use the repo-owned publish helper rather than a push-only path so version
  surfaces, generated plugin manifests, release artifact, tag, and GitHub
  release are kept in one ordered flow.
- Run the declared release gate after bump/sync:
  `./scripts/run-quality.sh --release`. This is the intended place for
  `release_only` copy-heavy tests to run.
- Pass this tracked critique artifact to the release helper so the release
  record names the review proof.

## Bundle Anyway

- Patch bump is the correct level. The delta is maintenance and quality
  hardening: gitignore-aware scan hygiene, disabled usage-episode adapter
  setup, release-only routing for copy-heavy tests, and handoff cleanup for
  post-release vocabulary and docs ergonomics.
- Current version surfaces agree on `0.7.8`, and `v0.7.9` does not exist before
  publish.
- Usage episodes remain intentionally disabled; the release should not imply
  runtime capture is enabled.

## Valid But Defer

- Usage-episode vocabulary remains a post-release product decision.
- Docs ergonomics still needs a README-first cleanup, with generated
  [docs/cli-reference.md](../../docs/cli-reference.md) treated as reference
  noise rather than a prose rewrite target.
- The release adapter still has no executable `requested_review_commands`.

## Over-Worry

- Do not block this patch release on enabling usage episodes. The adapter is
  deliberately present but disabled, and the handoff carries the vocabulary
  decision forward.

## Decision

Proceed with `v0.7.9` through the repo-owned publish helper after the tracked
critique commit is in place. The release record must preserve the disabled
usage-episode state and the release-only copy-heavy test intent.
