Date: 2026-06-20

## Decision Under Review

Cut a minor release charness 0.52.6 → 0.53.0 publishing the full unreleased
stack (v0.52.6..HEAD). The payload is larger than this goal alone: it bundles
the skill-body redesign (S0–S4: 18 cured public bodies + the new
`check_skill_cut_safety.py` pre-cut authoring check) AND the previously-committed
but unreleased north-star Phase-4 boundary work (new `issue` closeout rung-1
floors, `release` distinct-channel publish floors, `achieve` floor-grammar
refactor + portability deleak) and the overhaul-sweep tail.

## Failure Angles

- Bump-rationale narrative mismatch: the release record could under-describe the
  payload if it names only the S0–S4 body redesign while the tag-to-HEAD range is
  35 commits including behavior-affecting new blocking floors on `issue`/`release`.
- New blocking gates could reject previously-valid closeouts if they were
  unconditional rather than presence-gated/optional.
- Token rename (`applied-restarted` → `instance-synced`) with no grandfathered
  alias could mis-resolve a downstream goal artifact.
- Dangling reference cites: the body cures replaced negative guardrails with
  positive heuristics that cite reference files; a typo'd cite would silently drop
  a behavioral rule.
- Surface-lock / stuck-at-0.52.6: a version-bearing surface left behind would ship
  an inconsistent install surface.

## Counterweight Pass

- Real blockers: none. Frozen-pin prompt refactor + additive guardrails +
  internal refactor; gate suite green at every commit.
- Over-worry dismissed: "minor might be breaking" (no removed/renamed relied-on
  capability — the only rename has zero literal consumers and a membership-based
  gate); "frontmatter routing shift" (zero `description`/`name`/`allowed-tools`
  changes); "new tool unsafe" (`check_skill_cut_safety.py` is stdlib + repo
  `runtime_bootstrap` only, a helper not a blocking gate). Minor is the
  conservative-correct bump (patch would be dishonest given the new floors).
