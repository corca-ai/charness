## BLOCKER (found, fixed before publish)

1. **Default-ON provider shell-out was undocumented for consumers.**
   `load_issue_source_config` defaults `enabled: True` and the handoff SKILL.md
   prose makes `--with-issues` the pickup default, so every consumer who runs
   `charness update` to 0.12.0 gets a chunker that shells out to their issue
   provider at every bare `/handoff` pickup — with no operator/update-surface
   mention. Release notes would be dishonest by omission.
   - **Fix applied:** `.agents/release-adapter.yaml` `update_instructions` now
     carries a `BEHAVIOR CHANGE` line — 0.12.0 turns issue-source ON by default,
     it degrades to doc-only on failure, the exact opt-out is
     `issue_source: {enabled: false}`, and the non-gh path is named as not yet
     live-proven. This renders into the published `latest.md` User Update Steps.

## CAUTIONS (surfaced, non-blocking)

- **Minor bump is honest.** Stage-script change is purely additive — `--input`/
  `-i` added, `--entries`/`--merge-proposal`/`--chunk` kept as aliases, stdin
  now default, no previously-required flag removed; `parse` gained a positional
  and kept `--handoff-path`. No invocation break. Default-on issue source is
  opt-out and degrades safely, so additive capability, not a major.
- **#249 — both gaps resolved.** Narrow input: `--with-issues` unions the live
  backlog + dedups cited issues (live-proven 14 listed -> 9 merged -> 5
  net-new). Narrow trigger: bare `/handoff` / `invoked_directly` fires, task
  directive bypasses (fixture-pinned). Both closeable.
- **#248 — resolved.** Uniform input + stdin + loud exit-2 failure across the
  three JSON stages; the exact `prepare_ranker_packet` repro now reads stdin.
- **Honesty of close.** Non-claims (non-gh stub-only; `invoked_directly` host
  wiring unproven e2e; cautilus skipped) are ergonomics/trigger-orthogonal and
  do not block resolution; they are carried into the close comments so the
  non-gh and host-wiring risks stay tracked. Neither issue needs to stay open.

## Verdict
