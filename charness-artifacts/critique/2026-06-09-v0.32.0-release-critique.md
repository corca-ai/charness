# Release critique — charness v0.32.0 (minor; #339)

Date: 2026-06-09

Fresh-eye satisfaction: satisfied — the v0.32.0 release decision was reviewed by a
fresh-eye reviewer in a separate agent context; verdict SHIP after the two flagged
blockers were resolved.

## Scope

The minor bump 0.31.0 → 0.32.0 publishing the already-merged, already-reviewed #339
work (portable residual/disposition ledger + the domain-blind proof-semantics
adapter + the proof-mismatch floor on achieve and issue `verify-closeout`). The
review judged the RELEASE decision (bump level, operator-note accuracy, version
drift, publish boundary, git state), not the #339 code (which had 4 prior fresh-eye
reviews + a green broad gate).

## Verdict

REVISE → resolved → SHIP. The reviewer confirmed the bump level and operator note,
and raised two mechanical blockers + one should-fix, all now addressed.

### Confirmed correct (verified against code)

- **Minor is the right bump.** The three floors are genuinely additive /
  grandfathered / inert-by-default: the residual-ledger floor is date-grandfathered
  (`RESIDUAL_LEDGER_RULE_DATE = 2026-06-10`, `is_residual_ledger_enforced`); the new
  `accepted-risk:`/`out-of-scope:` arms only accept more and are EXCLUDED from the
  #337 destination vocabulary (`evaluate_destination_form`); the proof-mismatch floor
  returns `problem: None` when no `## Proof Ledger` is present. No existing artifact
  changes verdict → not major, and real new capability ships → not patch.
- **Operator note accurate.** "installed-plugin consumers inherit no new blocking
  behavior on existing artifacts or on closeouts that declare no proof ledger" holds
  against the code.
- **No version-surface drift.** `packaging/charness.json` is the single source;
  `sync_root_plugin_manifests.py` regenerates all five surfaces; `ensure_release_surface`
  hard-fails on residual drift before push.
- **Publish boundary honest.** `public_release_verification` defaults to
  "not checked by this helper" and only flips to verified after `gh release view`
  returns 0; the auto install-refresh never aborts the published release.

### Blockers — resolved before publish

- **[BLOCKER] Uncommitted adapter note** (`.agents/release-adapter.yaml` 0.32.0
  entry) → the publish refuses a dirty worktree. Resolved: committed with intent in
  the release-prep commit (this critique + the adapter + the doc note are committed
  before `--execute`).
- **[BLOCKER] Local `main` 2 commits ahead with the unrelated module-split DRAFT
  goal** (`2026-06-09-achieve-closeout-module-split.md`) → it would ship in v0.32.0.
  Resolved by DISCLOSURE (not removal): the goal is an inert, artifact-only
  `Status: draft` plan, and shipping an inert draft goal in a release is the
  established precedent (0.31.0 shipped the inert #339 draft). The 0.32.0
  `update_instructions` now names it explicitly, so its inclusion is intentional and
  honest, not silent.

### Should-fix — resolved

- **[SHOULD-FIX] charness ships no proof-semantics adapter**, so its own future
  `## Proof Ledger` closeouts run the proof-mismatch floor in degraded mode (every
  row needs a disposition). Resolved: a maintainer heads-up was added to the 0.32.0
  `update_instructions` and `docs/proof-semantics-adapter.md`.

### Over-worries (raised, not blocking)

- Version-surface drift and publish-boundary honesty — both verified already handled
  by the helper (see "Confirmed correct").
