Date: 2026-06-14

Standalone release critique (bounded fresh-eye, high-leverage tier, separate
agent context — Explore subagent `release-reviewer`) of the decision to publish
charness 0.49.0. The code was already critiqued in
`2026-06-14-pry-quality-integration.md` and

phase newly fail a clone/CI after publish?"

## Decision Under Review

Publish 0.48.0 → **0.49.0 (minor)** with two commits: (1) `2a3b5c4a` pry
testability backlog consumed by `quality` + agent-runtime spawn seam; (2)
`cd4ac5bd` track the upstream pry support skill (`support_skill_source`,
`external_binary_with_skill`) + auto-run the `inventory-testability-surface`
advisory phase + backlog-classification bug fix. Plus a 0.49.0
`update_instructions` entry in `.agents/release-adapter.yaml`.

## Failure Angles

- Wrong bump level (should it be patch, or does anything force major?).
- Something half-baked ships (does the release require the deferred upstream-skill
  materialization to function?).
- update_instructions overstate/understate operator impact.
- The new standing-gate phase newly fails a clone or CI without pry.
- Public-visibility / supply-chain risk in the install path.

## Counterweight Pass

- Minor is correct: purely additive (new advisory phase, new inventory, new
  manifest field, an injectable seam with default-preserved behavior); zero
  invocation/compatibility break, so not major; the new capability rules out patch.
- Nothing half-baked: the quality phase calls the `pry` binary only; it never
  imports the materialized `support/pry` skill, and degrades (exit 0) without the
  binary. Materialization is deferred to `charness tool sync-support`, matching the
  specdown/cautilus precedent.
- update_instructions accurate: correctly states the phase degrades exit 0 without
  pry, prints a non-empty backlog as ADVISORY, ships no githooks, and runs only
  inside `run-quality.sh`.
- Clone/CI safe: exists-guarded phase, always exit 0, bounded 120s timeout; a
  fresh clone without pry hits the degraded path and standing quality stays green.
- Supply-chain acceptable: HTTPS cargo-dist `releases/latest` installer for a
  first-party upstream (corca-ai/pry), v0.1.0 published.
