# Release Critique: v0.20.0

Fresh-Eye Satisfaction: parent-delegated

## Change

Publish the accumulated local Charness changes since v0.19.0 as v0.20.0 and
close #299 and #300 through the release carrier.

## Findings

### Act Before Ship

- Patch release would be too small for this delta. #299 adds an advisory
  quality inventory, #300 adds a setup host-docs-only helper, and closeout proof
  behavior changed. The release should be minor: v0.20.0.
- Release critique evidence was required by the publish helper. This artifact
  satisfies that release-scope review gate.
- `.agents/release-adapter.yaml` update instructions still described v0.19.0.
  They were updated before publish to describe v0.20.0 and the new operator
  surfaces.

### Bundle Anyway

- Real-host proof is triggered by the aggregate v0.19.0..HEAD delta through the
  integrations/control-plane surface. The release should carry that checklist
  as real-host proof still owed unless executed on a second machine or clean
  temp-home.

### Over-Worry

- Patch-vs-minor is not ambiguous after #299/#300: both are additive
  operator-facing capabilities, not only bug fixes.
- Broad pytest should not be re-run as a pre-lock closeout habit, but release
  quality may still run the configured release gate through the helper.

### Valid But Defer

- Maintained Cautilus scenario expansion for the setup helper was considered
  but not required for this release; deterministic tests cover the new helper
  path directly and dogfood records the scenario-registry decision.

## Evidence

- Fresh-eye reviewer Lovelace returned NO-GO for a patch release and recommended
  the minor release shape.
- Fresh checkout probes passed:
  `./charness --help`, `./charness goal check --help`, and
  `python3 scripts/doctor.py --repo-root . --json --skip-release-probe`.
- `current_release.py` reported version surface agreement at 0.19.0 before the
  release bump.
