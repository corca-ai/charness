## Scope

Standalone release critique before publishing `v0.14.0`.

## Angles

- Release surface, version level, and public-boundary correctness.
- Operator update text and validation economics.
- Counterweight pass over release blockers and over-worry.

## Counterweight Triage

### Act Before Ship

- Fixed: release update instructions must name `0.14.0` and the real operator
  impact. The adapter text is copied into `charness-artifacts/release/latest.md`
  by the publish helper, so stale `0.13.5` instructions would mislead installed
  users.
- Fixed: public GitHub release notes must be self-contained. This release uses
  `charness-artifacts/release/2026-06-02-v0.14.0-notes.md` rather than relying
  on generated commit notes that could omit `tokei` and broad-lock behavior.
- Fixed: release-time real-host checklist must match the changed integration
  surface. The unreleased delta includes `integrations/tools/tokei.json`, so
  the checklist now verifies `tokei` doctor/install/sync behavior instead of
  carrying stale Cautilus-specific proof.

### Bundle Anyway

- Added fresh-checkout probes for top-level CLI help, `charness goal check`
  help, and doctor JSON. This gives the new stable operator surface checkout
  shape proof before tag push.

### Valid But Defer

- Add a semantic release audit that checks update instructions mention the
  target version and release-specific hard dependencies. For this release, the
  practical blocker is fixed directly in the adapter and public notes.
- Broaden broad-pytest command-shape detection only if another canonical broad
  command family appears. Current detector matches the standing closeout command.
- Refresh quality runtime economics after `v0.14.0`; it is trend debt, not a
  release blocker.

### Over-Worry

- Minor bump is justified because `charness goal check` is a new additive
  operator-facing command and the validation behavior is meaningful but
  migration-free.
- A second-machine Cautilus proof is not the right proof for this slice. The
  configured real-host proof should cover the changed `tokei` integration.

## Result

No remaining release Act Before Ship findings after the adapter, public notes,
fresh-checkout probes, and `tokei` real-host checklist are included.
