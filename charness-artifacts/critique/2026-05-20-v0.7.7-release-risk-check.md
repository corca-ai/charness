# v0.7.7 Release Risk Check

Date: 2026-05-20
Scope: publish the full `v0.7.6..HEAD` maintenance delta as `charness` patch release `v0.7.7`.
Fresh-Eye Satisfaction: parent-delegated release critique angles plus a targeted post-fix reviewer.

## Act Before Ship

- The release helper must evaluate real-host proof against the previous released
  tag delta, not only `origin/main..HEAD`. This was fixed before release by
  deriving release content paths from `refs/tags/v<previous_version>..HEAD`
  when that tag exists, fetching the exact remote previous-version tag when the
  local tag is missing, and adding regression tests for already-pushed branch
  and missing-local-tag cases. A release-gate retry also hardened
  `--publish-current` so a failed post-bump publish can infer the previous
  semver tag instead of falling back to the pushed branch diff.
- Release notes and closeout must describe the whole `v0.7.6..HEAD` delta,
  not only the final runtime-hygiene commit.
- Run the declared release gate after bump/sync:
  `./scripts/run-quality.sh --release`.

## Bundle Anyway

- Patch bump is the right release level. The delta is maintenance and hardening:
  mutation sampling and score semantics, testability guidance, external-runtime
  hygiene, bounded helpers, and release proof repair.
- `v0.7.7` tag and GitHub release do not exist before publish; `v0.7.6` is the
  verified previous public release.
- `fresh_checkout_probes` remain intentionally unconfigured for this release;
  the real-host checklist is still carried when the actual tag delta hits the
  configured install/control-plane seams.

## Valid But Defer

- The release adapter still has no `requested_review_commands`. This release
  passes an explicit tracked critique artifact, but future release policy should
  configure an executable review gate or an explicit waiver path.
- Full second-machine proof is not automated. The release artifact must carry
  the adapter checklist instead of claiming local CI replaces host proof.
- `publish-current` now has a regression test for pre-bumped releases, but the
  normal documented release path remains `--part patch`.

## Over-Worry

- No generated manifest/version drift is present before the bump.
- Keeping issue `#183` open is correct; this release hardens the surrounding
  testability and mutation infrastructure but does not claim that broader watch
  item is closed.

## Decision

Proceed with `v0.7.7` only after the release-helper regression test, synced
plugin export, and full release gate pass. The release artifact must record
real-host proof as required for the actual tag delta.
