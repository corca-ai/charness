## Scope reviewed

## BLOCKER (found, fixed before publish)

1. **Operator-update steps would overclaim that `charness update` enables
   session-start routing.** `scripts/host_hook_install_lib.py` only wires the
   usage-episodes SessionStart hook (`HOOK_SCRIPT_RELATIVE` is hardcoded to
   `usage_episode_session_start.py`); nothing wires `session_start_find_skills.py`.
   So `charness update` to 0.11.0 delivers the script but does NOT enable the
   routing trigger — that needs a manual user-level hook.
   - **Fix applied (commit 90e92aa):** `.agents/release-adapter.yaml`
     `update_instructions` now states the find-skills hook is a manual
     user-level install + carries the honest non-claim; `find-skills`
     `references/session-start-routing.md` clarifies the user-level install.

## WATCH (found, fixed)

1. **`session-start-routing.md` named config paths without the user-level
   qualifier** (written before the repo→user move) — inconsistent with the
   script docstring. Fixed in 90e92aa.

## OVER-WORRY (raised, dismissed with reason)

- **Version drift across packaging/plugin/marketplace surfaces** — non-issue;
  `bump_version.py` runs the sync command and `ensure_release_surface` aborts on
  any drift.
- **"Published" claimed at tag push** — non-issue; the publish flow is
  synchronous and self-verifying (`verify_release_visible` hard-fails if the
  GitHub release is not visible).
- **Stale `latest.md` (0.10.0)** — non-issue; the helper rewrites it to the new
  tag and the narrative audit enforces the target tag is present.
- **Runtime state / mid-edit artifacts shipping** — non-issue; `.charness/` is
  gitignored, worktree clean, the net `origin/main..HEAD` diff is intended.

## Bump verdict

**MINOR (0.11.0)** — new additive operator capability (find-skills routing
script + skill contract), no compatibility break; #238/#239 prose fixes ride
along. PATCH understates a new mechanism; MAJOR is unwarranted.

## Decision

Proceed with publish (MINOR). Both pre-publish findings fixed and committed.
Follow-up worth filing: extend `host_hook_install_lib` to optionally
auto-install the find-skills trigger (parallel to usage-episodes) so it is not
