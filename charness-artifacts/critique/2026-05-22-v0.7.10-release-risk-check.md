## Release Scope

- Version: `0.7.10`; tag: `v0.7.10`.
- Consumer change: fail-closed release and quality discovery, stricter repo file
  listing gates, and clearer CLI progress/update notices.
- Review scope is the committed release delta `v0.7.9..HEAD`; the packet's
  clean-working-tree report is only the publish precondition snapshot.

## Surface-Lock Inventory

- Generated install surfaces: packaging manifest, Claude/Codex plugin manifests,
  marketplace metadata, and synced plugin export.
- Consumer-visible behavior: `charness update` notice suppression during active
  update, closeout heartbeat progress dots, and fail-closed release/quality
  diagnostics.
- Documentation and artifacts: README/workflow routes, quality latest pointer,
  release latest pointer, debug/retro/critique records.
- Adapter and integration surfaces: release real-host proof checks, quality
  standing gates, and repo file listing strict mode.

## Act Before Ship

- `./scripts/run-quality.sh --release` must pass. Fresh-eye review found the
  current quality pointer exceeded the 140-line limit by one line; the release
  gate cannot be bypassed.
- The critique artifact and prepare packet must be tracked before invoking the
  publish helper, because `publish_release.py` requires a clean worktree and a
  tracked `--critique-artifact`.
- The release record for `v0.7.10` must be regenerated by the publish helper
  before tag push; the previous `latest.md` still names `v0.7.9`.
- The release artifact must carry the configured real-host proof checklist
  because `v0.7.9..HEAD` touches `README.md` and integrations/control-plane
  seams.

## Bundle Anyway

- Patch is the right bump level. The delta is maintenance hardening and operator
  clarity, not a compatibility break.
- Fresh-checkout probes are not configured; the release artifact should say that
  explicitly rather than inventing extra proof.
- Requested-review enforcement remains advisory-only in the release adapter; the
  delegated critique artifact is the release review proof for this slice.

## Over-Worry

- Do not block this release on broad documentation polish or making
  `charness update` emit heartbeat dots. The update path already exposes phase
  markers and suppresses the stale "run update" notice while updating.

## Valid But Defer

- Improve release critique packets so they can render a release range like
  `v0.7.9..HEAD`, instead of only reporting the dirty working tree.
- Tighten `cli_skill_surface_skill_paths` in the release adapter after this
  release; the current surface check reports no blocker.

## Operator Action Required

- Run the repo-owned publish helper with this tracked critique artifact.
- After publish, run `charness update`; restart Claude Code or Codex if the host
  cache still shows the previous version.

## Decision

Proceed only after the quality line-budget blocker is fixed, the critique files
are committed, and the publish helper rewrites the `v0.7.10` release artifact
before tag and GitHub release publication.
