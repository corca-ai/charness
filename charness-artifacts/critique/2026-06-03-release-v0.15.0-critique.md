# Release Critique: v0.15.0

Fresh-Eye Satisfaction: parent-delegated

## Target

Release `charness` after #280 introduced the Corca-internal usage
product-review reporter.

## Change

Publish a minor release from `0.14.0` to `0.15.0` carrying commit `e652d227`
and release closeout for #280.

## Act Before Ship

- Stale operator update text would have told users to pull `0.14.0` in the
  `0.15.0` release artifact. Disposition: applied in
  `.agents/release-adapter.yaml`; the first update instruction now names
  `0.15.0` and the new product-review reporter.
- A patch bump would understate the user-visible effect. The change adds a new
  maintained reporter script in root and plugin surfaces, so the repo
  `version-policy.md` maps it to a minor release. Disposition: publish with
  `--part minor`.

## Bundle Anyway

- `current_release.py` reported no version drift before release mutation:
  packaging, Claude plugin, Codex plugin, and marketplace surfaces were aligned
  at `0.14.0`.
- `v0.15.0` is the intended target after applying the version-policy finding.
- #280 is open and reachable through `gh`; release helper will preflight and
  verify `--close-issue 280`.
- Fresh-checkout probes are configured and passed in read-only preflight.

## Over-Worry

- Seeing all surfaces at `0.14.0` before publish is expected; the release helper
  owns bump and sync.
- No release-time real-host proof trigger matched the #280 script/doc slice;
  this is consistent with the adapter's configured real-host path globs.

## Valid But Defer

- Generated GitHub notes are acceptable for this release. A hand-authored notes
  file could be clearer, but the release helper validates the public release
  narrative and issue closeout carrier.
- #184 remains open by design and should not be implied closed by this release.
