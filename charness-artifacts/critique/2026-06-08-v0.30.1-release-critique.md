## Decision Under Review
Ship `charness` 0.30.1 (patch) for a single additive, opt-in release-flow change: the maintainer install-refresh (`charness update`) is now AUTO-RUN by `publish_release.py` after a verified publish (adapter-declared `post_publish_install_refresh`, non-blocking, on normal and `--resume` paths). This publish dogfoods the new behavior on the authoring machine.

## Failure Angles
- **Bump dishonesty.** Patch could hide a breaking flow change or a new capability that deserves minor.
- **Stale update_instructions / staleness-guard HOLD.** The 0.30.1 entry could fail to mention the target version, tripping the round-1 HOLD; or it could overclaim.
- **Adapter no longer loads.** A new field plus a long prose entry could break adapter parsing and brick the publish helper that consumes it.
- **Dogfood blast radius.** Auto-running `charness update` on the live authoring machine could mutate the wrong host, or a failed/hung refresh could abort an already-published release.
- **Resume gap regression.** The prior critique flagged a resume-path miss; a regression would silently skip the refresh on `--resume`.
- **Issue-state contamination.** The diff could re-open or wrongly touch #336 / issue closure.
- **Overclaimed proof.** The entry could assert live/provider/real-host proof where only local ran.

## Counterweight Pass
Most angles are answered by executed evidence, not assertion. The adapter loads cleanly under the repo's own hand-rolled parser (`scripts/adapter_lib.load_yaml` — NOT PyYAML; a plain `yaml.safe_load` ScannerError at line 34 is a false alarm, since every prior shipped entry carries the same `` `field`: value `` colon pattern and v0.30.0 shipped on it). The staleness guard returns `None` for target 0.30.1 (verified against the live adapter via the real loader). All 20 resilience tests pass, including end-to-end auto-run, `not_configured` skip, and `failed`-without-raising. The resume path IS wired (`publish_release_resume.py`). The only installed-host mutation is the single opt-in adapter command on the authoring machine — exactly the intended dogfood target. No real angle survives as a release blocker; the YAML-parser scare is over-worry once the repo's actual loader is used.
