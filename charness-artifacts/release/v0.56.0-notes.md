# Charness v0.56.0

Minor release for quality-review efficiency and stronger public-web gathering
fallbacks.

## Changes

- Adds explicit `web-fetch` fallback stages for optional `curl_cffi`
  impersonated requests and headless Patchright rendering/network diagnostics.
- Keeps blocked fetches honest: HTTP `403+` responses are classified as
  failed/error-page attempts instead of weak success, even when the body is long.
- Adds compact YAML/summary views for quality ergonomics and standing test
  economics so agent review can consume advisory inventories with fewer tokens.
- Refines standing pytest execution so release-only nested CLI tests stay out of
  routine runs and worker selection stays bounded.

## Operator Notes

- No migration is required.
- Run `charness update` to install the latest published release.
- Existing active agent sessions should be restarted or refreshed before relying
  on the updated installed skill text.
- `curl_cffi` and Patchright are optional Python/runtime dependencies for the
  new fallback routes. `charness update` does not install them as managed
  integration tools; when absent, `web-fetch` records skipped `missing-tool`
  attempts and continues through the remaining route plan.

## Non-Claims

- This release does not guarantee retrieval from sites that actively block
  automated access.
- Patchright fallback stays headless and read-only; it does not click, submit
  forms, authenticate, or solve challenges.
- Compact quality summaries are triage/readability surfaces, not stronger proof
  than the underlying validators and full JSON inventories.
