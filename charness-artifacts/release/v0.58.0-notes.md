# charness v0.58.0

## Scope

Claim-fidelity floor overhaul (reference-compaction Slices 1–7 plus follow-ups).
Additive skill-authoring and eval surfaces only — **no invocation contract
changed, no migration**.

## Changes

- **Closeout Vocabulary surfaces.** `impl`/`setup`/`spec`/`create-skill` now
  define an emittable `## Closeout Vocabulary`, so skill closeouts state canonical,
  honest status tokens instead of a vague "done" (e.g. `setup` normalization emits
  `Repo mode:`, a per-surface status, and `Normalization non-claims:`).
- **Claim-fidelity floor model.** Evals moved from doc-open floors
  (`requiredCommandFragments`) to observed emitted-token / substance floors
  (`requiredSummaryFragments` + `outcome-assertions.json`) wherever a live capture
  proved the doc-open was a refuted proxy — re-baselined per skill by ask-before-run
  captures (impl, setup, handoff, gather, hotl, critique, hitl, achieve, create-cli).
- **Reference compaction.** Removed duplicated/dead skill reference docs; added
  advisory `classTag` annotations; made `## Closeout Vocabulary` headroom-exempt.
- **handoff planner.** Pickup now reads `continuation-sequence.md` only when the
  pickup is genuinely ambiguous (several plausible pickups), not unconditionally.
- **eval runtime fix (#409).** The floor matcher reads the closeout from the
  authoritative stream, not the truncated session tree.

## Behavior notes

- No operator-typed command, flag, doctor output, or install step changed.
- The only consumer-observable behavior shifts are the `handoff` pickup read
  condition and the `setup` normalization closeout vocabulary — both
  additive/clarifying.

## Resolves

- **#413** — setup claim-fidelity floor redesigned as an artifact/substance
  instrument, proven by a live capture (old doc-open floor FAILED / new
  emitted-token floor PASSED).
- **#412** — handoff pickup floor split into per-condition falsifiable fixtures
  (clear + ambiguous), both capture-proven.

## Accepted precondition

- **#404** (scheduled full / JS StrykerJS mutation regression) remains open; the
  release quality gate does not depend on full-mutation green.

## Install

Run `charness update` to install this release; then `charness doctor` to confirm
no version skew.
