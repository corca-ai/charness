# Charness v0.55.1

Patch release for the #401 public-skill quality closeout.

## Changes

- Fixes `critique` packet ownership so standalone critique runs prepare packets
  when adapter sections exist, while parent-produced packets remain supported.
- Records the completed `spec` and `impl` quality-led skill review, including
  evidence-backed no-change decisions.
- Keeps the #401 quality evidence explicit: `retro`, `critique`, `spec`, and
  `impl` now all have per-skill assessments.

## Operator Notes

- No migration is required.
- Run `charness update` to install the latest published release.
- Existing active agent sessions should be restarted or refreshed before relying
  on the updated installed skill text.

## Non-Claims

- This release does not add live evaluator-backed dogfood proof for `spec` or
  `impl`; that remains a follow-up when a concrete behavior log or regression
  prompt exists.
