# Charness v0.56.4

Patch release focused on test and script execution efficiency plus closeout
evidence hygiene. The update path is unchanged: run `charness update` after the
release is public. No data migration is expected.

## Highlights

- Faster test smokes: repeated script-entry tests now call repo-owned `main()`
  functions in process through a shared `tests.script_main.run_loaded_script_main()`
  helper instead of launching extra Python interpreters.
- Cleaner boundary inventory: raw boundary candidate keys dropped from the
  high 80s during the round to 67, and clean-convertible files dropped to 3
  before release proof.
- Safer conversion discipline: generated/exported command tests, web-fetch
  browser cleanup, Cautilus eval boundaries, release-host proof, and committed
  packaging checks remain subprocess-backed where the process boundary is the
  behavior under test.
- Closeout evidence: the round-3 goal now records a host metric window, host
  probe JSON, retro, and disposition review before release publication.

## Compatibility

- No public skill, package id, command name, or install surface is renamed.
- The shared test helper affects repository tests only; public runtime behavior
  is unchanged.
- Existing operator update instructions remain valid.

## Operator Notes

- Run `charness update`.
- Restart active Codex or Claude sessions if the update reports rotated plugin
  cache paths.
- The release does not require a manual migration or rollback procedure.
