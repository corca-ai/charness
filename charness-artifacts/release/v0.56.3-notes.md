# Charness v0.56.3

Patch release focused on quality, speed, and release-closeout reliability. The
update path is unchanged: run `charness update` after the release is public.
No data migration is expected.

## Highlights

- Faster standing proof: smoke eval scenarios now run with bounded parallelism
  by default, while `--jobs 1` preserves the previous sequential path.
- Faster quality scripts: the standing Specdown phase now runs spec files with
  file-level jobs, and several timeout tests avoid fixed one-second waits.
- More precise timeout controls: startup probes, Cautilus eval forwarding, and
  the generic script timeout helper accept positive fractional seconds while
  remaining compatible with integer values.
- Update flow efficiency: `charness update all` reuses precomputed support-sync
  results instead of repeating the same support-sync work.
- Closeout/release proof hygiene: the release critique records the full
  `origin/main..HEAD` release range, final release proof still runs through the
  repo-owned release helper, and fresh-checkout probes remain required before
  publish is claimed.

## Compatibility

- No public skill, package id, or install surface is renamed.
- Existing integer timeout values keep working.
- Operators can opt into sequential eval smoke runs with `--jobs 1` when needed.

## Operator Notes

- Run `charness update`.
- Restart active Codex or Claude sessions if the update reports rotated plugin
  cache paths.
- The release does not require a manual migration or rollback procedure.
