# charness 0.17.0

This release makes the testability quality ratchet part of the public quality workflow and leaves the next work queue explicit in `docs/handoff.md`.

## What's New

- The quality workflow now tracks the boundary-bypass ratchet as a standing signal: candidate counts, clean-convertible/internal classification, keep-boundary decisions, and schema drift are validated by repo-owned scripts.
- `find-skills` routes testability, test DSL, lazy eval, and boundary-bypass-ratchet requests toward `quality`, so the same path is easier to reuse in downstream Charness repos.
- Release and quality artifacts now carry the completed testability-ratchet proof instead of leaving the old goal active.
- The next session is queued for #284 first, then #286, with #285 only if the #286 fixture work exposes live-issue brittleness.

## Operator Notes

- Run `charness update` to pull 0.17.0.
- Restart Claude Code or Codex if the host cache still shows the previous version.
- No manual migration is required beyond the normal update flow.
- Runtime budget for `check-duplicates` on the local x86_64 profile is now 13.0s, matching the current release-path workload while staying a narrow enforced budget.

## Carried Forward

- 0.16.0 provider-safe `achieve`/`retro` closeout metrics remain available.
- `validate_quality_artifact.py --report-all` remains available and default behavior stays fail-fast.
- Stable `charness goal check`, broad closeout verification lock, and `tokei`-backed Python length gates remain part of the installed surface.
