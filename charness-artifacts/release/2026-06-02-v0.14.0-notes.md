# Charness 0.14.0

This minor release adds a stable goal-helper command and tightens closeout
validation so expensive broad pytest runs happen only after the mutation set is
locked.

## Operator Impact

- `charness goal check` is now the stable operator surface for goal readiness
  checks. Use it instead of invoking helper scripts from a versioned plugin
  cache path.
- `scripts/run_slice_closeout.py` now refuses broad pytest unless
  `--verification-lock` is passed after the mutation set is locked. Use
  `--skip-broad-pytest` for pre-lock rehearsal of deterministic gates.
- Python file and headroom length gates now use `tokei` Python code-line counts
  and fail closed when the `tokei` binary or JSON output is unavailable.
  Function limits remain AST-span based.
- The broad-lock, `tokei`, and stable goal-helper lessons are now recorded in
  repo operating guidance and generated plugin/export surfaces.

## Update Steps

Run `charness update` to pull 0.14.0, then restart Claude Code or Codex if the
host cache still shows the previous version.

Validation operators should ensure `tokei` is available on PATH before running
quality gates. Use `charness tool doctor tokei --json` or
`charness tool install tokei --json` on machines that do not already have it.

No manual migration is required beyond the normal update flow.
