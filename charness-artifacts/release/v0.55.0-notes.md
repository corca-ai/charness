# Charness v0.55.0

## Operator Action

Run `charness update` to install the latest published Charness release.

No migration is required. Already-running Codex or Claude sessions may continue
using the skill cache they loaded at startup; start a new session after updating
when you need the v0.55.0 skill behavior immediately.

Rollback path: reinstall or pin the previous published release, `v0.54.2`, and
restart any session that should use the older skill cache.

## Scope

This release tightens the skill-improvement loop around `quality` and `retro`
and ships the helper/validator changes needed to keep those improvements
machine-checkable.

- `quality` now separates target-skill review from ambient repo gate state more
  clearly and validates explicit skill ergonomics count claims against the
  measured count.
- `retro` now has prepare-packet support and requires future/current session
  retros to record machine-readable persistence as `Persisted: yes: <path>` or
  `Persisted: no: <reason>`.
- Release update instructions are evergreen; release-specific migration,
  rollback, and operator expectations belong in the public release notes.
- Shared artifact validator helpers now keep changed-path selection consistent
  across critique, ideation, and retro validators.

## Verification

The release candidate passed the repo quality gate before publication:
`./scripts/run-quality.sh --read-only` reported `79 passed, 0 failed`.

Changed-line mutation coverage was also checked against `origin/main` with the
reused coverage artifact and reported `ok: true`.

## Compatibility

Existing public skill names and CLI entrypoints are unchanged.

Historical retro artifacts remain valid under the grandfathering window. Retro
artifacts dated on or after 2026-06-25 must use the new persisted-status form.

## Release State

The release helper publishes the branch and tag, verifies the public GitHub
release, records a distinct-channel release check, and then runs the
adapter-declared post-publish install refresh on the maintainer machine.

The automatic refresh is local to this maintainer machine. It does not update
already-running sessions or every operator machine.
